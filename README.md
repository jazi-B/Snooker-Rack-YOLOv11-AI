# 🎱 Real-World Snooker Rack Detection System (YOLOv11)

A complete, production-grade Computer Vision pipeline to detect the **Snooker Rack (Triangle Formation)** from a **fixed overhead CCTV camera**. 

Built with **Ultralytics YOLOv11**, OpenCV, and an **Iterative Pilot-First Industry Workflow**.

---

## 💡 Industry Best-Practice Strategy: Pilot Model First

As specified in real-world computer vision engineering:
> **Do not annotate 150–300 images upfront.**  
> Always start by annotating **20–30 images** to train a **Pilot Model**.

### Why the Pilot Approach Saves Days of Work:
1. **Labeling Boundary Verification**: Ensures your bounding box definition is precise (e.g. whether to include the wooden frame of the rack vs just the tight 15-ball triangle).
2. **Pipeline Testing**: Validates folder structure, script execution, CUDA acceleration, and export formats before heavy data labeling.
3. **Early Failure Discovery**: Catches lighting glare issues, occlusion edge cases, and distance scale problems early.

## 🛠️ Step-by-Step Implementation Guide

### Step 1: Image Collection Options

#### Option A: Own CCTV Camera (Best for Production)
Extract 20–30 frames from your fixed overhead CCTV camera.

#### Option B: Web & Public Datasets (If you don't have camera images yet)
You can start immediately using web images or public Roboflow Universe datasets!

1. **Download Web Samples using our helper script**:
   ```bash
   python src/download_web_samples.py
   ```
2. **Or search Roboflow Universe**:
   - Go to [universe.roboflow.com](https://universe.roboflow.com)
   - Search for `"snooker rack"`, `"snooker triangle"`, or `"snooker table top down"`
   - Download pre-annotated dataset directly in **YOLOv8 / YOLOv11 PyTorch TXT** format!


---

### Step 2: Pilot Annotation (Roboflow or CVAT)
1. Open **Roboflow** or **CVAT.ai**.
2. Create project: **Single Class Detection**: `snooker_rack`.
3. **Annotation Standard**:
   - Draw a **tight rectangle** surrounding the 15-ball racked triangle (and wooden/plastic triangle rack if included).
   - Keep bounding boxes crisp touching ball edges.
4. Export annotations in **YOLOv8/YOLOv11 PyTorch TXT format**.

---

### Step 3: Dataset Preparation & Validation
Place your exported images in `raw_data/images` and labels in `raw_data/labels`, then run:

```bash
python src/dataset_prep.py
```
This automatically splits data into `dataset/images/train`, `dataset/images/val`, `dataset/labels/train`, and `dataset/labels/val` and validates bounding box coordinates (0.0 to 1.0 range).

---

### Step 4: Phase 1 - Train Pilot Model (20–30 Images)
Train a fast nano model (`yolo11n.pt`) for 30–50 epochs:

```bash
python src/train_pilot.py
```

#### Pilot Audit & Verification Checklist:
- Inspect output images in `runs/detect/snooker_rack_pilot/val_batch0_pred.jpg`.
- Ask:
  - Is the bounding box hugging the triangle tightly?
  - Are loose balls mistaken for a rack?
  - Does the model detect the rack across different table ends?
- **Decision**: If any issue is found, adjust your annotation guidelines NOW on the 20 images rather than fixing 300 later!

---

### Step 5: Phase 2 - Expand Dataset (150–300 Images)
Once the pilot model passes validation:
1. Capture & annotate 150–300 total images.
2. Include negative samples (empty table with balls scattered, break shot in action, rack removed).
3. Re-run `dataset_prep.py`.

---

### Step 6: Train Full Production Model
Train using `yolo11s.pt` or `yolo11m.pt` with domain-specific augmentations (HSV saturation, brightness tolerance for CCTV glare):

```bash
python src/train_full.py
```

---

### Step 7: Evaluate Metrics
Check Precision, Recall, and mAP:

```bash
python src/evaluate.py
```

---

### Step 8: Live CCTV Stream Inference & Deployment
Run real-time inference on live RTSP CCTV camera feed or recorded video:

```bash
# Live USB / Camera feed (Index 0)
python src/inference_live.py --source 0 --conf 0.5

# Live RTSP Stream from CCTV camera
python src/inference_live.py --source "rtsp://admin:password@192.168.1.100:554/stream1" --conf 0.55

# Pre-recorded video file
python src/inference_live.py --source "match_video.mp4" --conf 0.5
```

## ⚡ Hardware & Performance Recommendations
- **Inference Speed**: YOLOv11n / YOLOv11s runs at **> 60 FPS** on NVIDIA GTX 1060 / RTX 3050 or **> 30 FPS** on CPU via ONNX Runtime / TensorRT.
- **ROI Optimization**: Since the camera overhead position is fixed, set ROI cropping in `inference_live.py` to process only the table region, saving 40% GPU/CPU overhead.
